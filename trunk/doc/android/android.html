A componente

A componente tem como mote a interface com o usuário para as respostas dos formulários editados na componente HTML5, a persistência local dessas respostas e a persistência via webservice Tomcat. É um aplicativo móvel Android.

 

Android

Sistema operacional para celulares adquirido pela Google, baseado no kernel Linux, foi desenvolvido pelos colaboradores (inclusive Google) da The Open Handset Alliance, chamado de Android Open Source Project. Sua arquitetura consiste em camadas bem definidas:

  • Aplicação: Onde rodam as funcionalidades e aplicações do celular;
  • Framework de Aplicação: Contém aplicações de suporte ao sistema, gerenciadores em geral. (Janelas, Contatos);
  • Bibliotecas: Bibliotecas de apoio ao funcionamento do sistema. (Open GL, SQLite, SSL);
  • Android Runtime: Contém a maquina virtual Dalvik e biblioteca adjacentes;
  • Linux Kernel: O kernel linux.

Máquina Virtual Dalvik

Escrita por Dan Bornstein, é o núcleo do Android, antes do sistema executar qualquer processo ele é convertido para linguagem dessa máquina e, depois, executado na mesma, otimizada para pouca memória e suporte a threatding.

 

Respostas

O aplicativo necessita ser configurado antes de proceder as respostas:

  • O usuário digita a URL retornada da persistência feita na componente HTML5;
  • A URL, através da componente Tomcat, retorna um XML, esse é convetido em um interface gráfica, possibilitando as respostas.

 

Após as conversões, o aplicativo está pronto para receber as respostas do usuário:

  • O usuário utiliza a interface gráfica para responder;
  • E as salva localmente.
